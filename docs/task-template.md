# Task Template

Before writing code, always do these 2 steps:

## Step A — Planning
First produce:
1. Intent
2. Scope
3. Files to change
4. Schema impact
5. Risks
6. Test plan

Do not write code until this plan is approved.

## Step B — Implementation
After approval:
- implement only the scoped task
- do not change unrelated files
- add tests
- summarize what changed
